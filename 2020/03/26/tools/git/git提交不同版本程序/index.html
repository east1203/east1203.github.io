git提交不同版本的程序

参考

https://blog.csdn.net/sinat_39150454/article/details/73729196

以前项目中版本控制通常使用小乌龟SVN版本控制器，最近发现github更胜一筹，不仅方便代码管理，而且还能反映程序员编程情况。今天简单记录一下如何使用git命令提交不同版本的项目代码到github.

1.查看版本分支

git branch -a #查看github现有的分支
git branch #查看本地分支
1
2
2.在本地创建分支
默认情况下，本地和远程端(github)只有一个名为master的分支。有时候为了方便区分每次修改提交的代码，我们可以通过利用不同的分支来加以区分，后续克隆clone代码可以方便的利用分支名签出不同版本的源程序。

git branch
1
3.切换当前分支

git checkout
1
4.推送新建分支到github上

git push origin
1
5.提交项目到github

git add .
git commit -a -m ‘YourComments’
git push -u origin
1
2
3
接着输入github账号和密码即可完成项目提交。
